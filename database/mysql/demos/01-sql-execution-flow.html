🚀 MySQL SQL执行流程

深入理解一条SQL语句的完整执行旅程

SELECT * FROM users WHERE id = 1;
客户端
→
连接层
→
解析器
→
优化器
→
执行器
→
存储引擎
1
客户端发起连接
Java应用通过JDBC驱动建立与MySQL服务器的TCP连接
// Java代码示例 Connection conn = DriverManager.getConnection( "jdbc:mysql://localhost:3306/test", "root", "password" );
2
连接层处理
MySQL连接器验证用户身份，分配连接线程，管理连接状态
-- 查看当前连接 SHOW PROCESSLIST; -- 连接数统计 SHOW STATUS LIKE 'Connections';
3
SQL解析
词法分析识别关键字，语法分析构建解析树，语义分析验证权限
词法分析: [SELECT] [*] [FROM] [users] [WHERE] [id] [=] [1] 语法树: SELECT -> (*, FROM users, WHERE id=1) 语义检查: 表存在性、列存在性、权限验证
4
查询优化
基于成本模型选择最优执行计划，决定是否使用索引
-- 查看执行计划 EXPLAIN SELECT * FROM users WHERE id = 1; -- 成本分析 SHOW STATUS LIKE 'Last_query_cost';
5
执行器调用
根据执行计划调用存储引擎接口，获取数据
执行计划: 使用主键索引查找 调用接口: handler->index_read_map() 返回结果: 1行数据
6
存储引擎处理
InnoDB引擎在Buffer Pool中查找页面，如未命中则从磁盘读取
1. 检查Buffer Pool缓存 2. 如果缓存未命中，从磁盘读取页面 3. 在页面中定位记录 4. 返回结果给执行器
7
返回结果
查询结果通过网络返回给客户端应用程序
// Java接收结果 ResultSet rs = stmt.executeQuery("SELECT * FROM users WHERE id = 1"); while(rs.next()) { System.out.println(rs.getString("name")); }
